08 / 08
Open questions
repo
How should the contracts pull in forge-std?
Foundry needs its test library (forge-std). The standard way is a git submodule under contracts/lib, which I've done, but it's the repo's first submodule, so a fresh clone needs git submodule update --init. Keep the submodule, or vendor it another way (Soldeer, or commit the files directly)?
contract
On-chain map vs off-chain resolve for "is this token spent?"
A spend points at a content-bound marker, not the plain token id, so a spent-check by id needs a lookup. I store a tokenID→marker map on-chain (one extra write per output) so isSpent(id) answers in one call. The alternative is a leaner contract where the driver does two calls. I chose the one-call version. Good tradeoff, or prefer the leaner contract?
EIP-712
Confirm the EIP-712 domain name "Panurus"
Every endorser signature is bound to an EIP-712 domain carrying a human-readable name; I set it to "Panurus". It's already baked into the frozen golden digests, so changing it later means re-freezing them. Keep "Panurus", or pick a different name now? (design §16)
week 4
Who is allowed to request an endorsement?
In Week 4 an endorser node needs to decide who may ask it to endorse. The planned default is the TMS network's own nodes (the EVM version of Fabric's MSP/ACL creator check). Flagging early so we can lock the policy before then. (design §16)
